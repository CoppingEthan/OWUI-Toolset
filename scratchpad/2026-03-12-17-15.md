# Title: Dashboard chart now shows cache_read tokens + three-mode toggle

## Issue Summary
Dashboard chart was showing `input_tokens + cache_creation_tokens` as "input" and `output_tokens` as "output", but completely hiding `cache_read_tokens`. Cache reads are often the largest token category (50-75% of total), so the dashboard showed dramatically lower numbers than Anthropic's console.

## Root Cause
The SQL query in `getTokenUsageTimeSeriesByDomainGroup()` only selected `SUM(input_tokens + cache_creation_tokens)` and `SUM(output_tokens)`. Cache read tokens were tracked in the database but never passed to the chart.

## Old Behaviour
- Dashboard chart showed ~1.9M tokens for a day when Anthropic billed 4.3M
- No way to see cache read tokens in the chart
- Two chart modes: Tokens (stacked input/output per domain) and Cost

## New Behaviour
- Chart has three modes via toggle buttons: **Tokens**, **Split**, **Cost**
- **Tokens** (default): Single bar per domain showing total = input + cache_write + cache_read + output. Numbers now match Anthropic's console.
- **Split**: Three stacked segments per domain — input (lightest), cache (mid), output (darkest). Tooltips show breakdown and totals for each category.
- **Cost**: Unchanged from before.
- Legend in Split mode shows one entry per domain; clicking toggles all three segments together.

## Files Changed
- **src/database/database.js:926** — Added `SUM(cache_read_tokens) as cache_read_tokens` to `getTokenUsageTimeSeriesByDomainGroup()` query
- **src/dashboard/public/index.html:1218** — Added "Split" toggle button between "Tokens" and "Cost"
- **src/dashboard/public/index.html:1417** — Updated `chartMode` comment to reflect three states
- **src/dashboard/public/index.html:1455-1461** — Added `cache` color tier (55% opacity) to `getDomainChartColors()`
- **src/dashboard/public/index.html:1465-1473** — Added 4.6 model labels
- **src/dashboard/public/index.html:1910-1990** — New shared callback functions: `chartTooltipLabel`, `chartTooltipAfterBody`, `chartLegendFilter`, `chartLegendClick` — handle all three modes
- **src/dashboard/public/index.html** — Updated `renderChart()` dataset creation: tokens mode builds single total bar, split mode builds three segments, cost mode unchanged

## Key Design Decisions
- Default mode is combined total (Tokens) rather than split, so the dashboard matches what the user sees in Anthropic's console at a glance.
- In split mode, color tiers use the domain's assigned color at different opacities: output=90%, cache=55%, input=25%. This keeps the domain identity clear while distinguishing token types.
- Shared callback functions avoid duplicating tooltip/legend logic between the "update" and "create" chart code paths.
