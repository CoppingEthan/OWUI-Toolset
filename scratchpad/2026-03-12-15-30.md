# Title: Bug fix, dependency updates, and dead code removal

## Issue Summary
Five changes made during a codebase audit:
1. **Bug fix**: `compressImage` was called but never imported — runtime crash on image upload via `/process`
2. **npm audit fix**: Resolved 6 of 14 dependency vulnerabilities (remaining 8 are unfixable in `markdown-pdf` and `xlsx`)
3. **SDK update**: `@anthropic-ai/sdk` upgraded from ^0.30.0 to ^0.78.0
4. **SDK update**: `openai` upgraded from ^4.67.0 to ^6.27.0 (Responses API fully compatible, verified)
5. **Dead code removal**: Stripped ~310 lines of unused transformer functions from `message-transformer.js`

## Root Cause (for bug fix)
`server.js` line 282 calls `compressImage()` for image uploads via the `/process` endpoint, but the import at line 19 only pulled `detectImageType` and `compressForLLM` from `image-compressor.js`. Any image upload through the CEE document loader would throw `ReferenceError: compressImage is not defined`.

## Old Behaviour
- Image uploads via `/process` crash with ReferenceError
- `@anthropic-ai/sdk` at 0.30.0 (missing newer prompt caching improvements)
- `openai` at 4.67.0 (old Responses API surface)
- `message-transformer.js` was 334 lines with 6 exported functions; only `isMultimodalContent` was imported anywhere

## New Behaviour
- Image uploads via `/process` work correctly — `compressImage` is now imported
- `@anthropic-ai/sdk` at 0.78.0 — constructor, `messages.create`, `messages.stream` all verified compatible
- `openai` at 6.27.0 — `responses.create` (streaming + non-streaming), constructor all verified compatible
- `message-transformer.js` is 20 lines exporting only `isMultimodalContent`

## Files Changed
- **src/api/server.js:19** — Added `compressImage` to the import from `image-compressor.js`
- **src/transformers/message-transformer.js** — Replaced entire file; removed `parseDataUrl`, `extractImages`, `extractText`, `transformToOpenAI`, `transformToAnthropic`, `transformToOllama`, `transformMessages` (all unused). Kept only `isMultimodalContent`.
- **package.json** — `@anthropic-ai/sdk` ^0.30.0 → ^0.78.0, `openai` ^4.67.0 → ^6.27.0
- **package-lock.json** — Regenerated by npm install + npm audit fix

## Key Design Decisions
- The dead transformer functions (`transformToOpenAI`, etc.) were from an earlier architecture where message formatting was centralized. Each provider now handles its own formatting internally (in `src/tools/providers/*.js`), making the centralized transformers redundant. Removed entirely rather than leaving commented-out code.
- Both SDK upgrades were verified for API compatibility before committing: Anthropic's `messages.create`/`messages.stream` and OpenAI's `responses.create` (streaming + non-streaming) all work identically.
- `deploy.sh` requires no changes — `npm install` at step 6 picks up the new versions from `package.json` automatically.
