# Title: V2 simplification — single-process, Ollama removal, provider abstraction, module split, security hygiene

## Issue Summary
The codebase had grown to ~20,000 lines with accumulated structural and security issues:
- Two processes (API + dashboard) sharing a SQLite file via a custom cross-process merge.
- Three LLM roles in `owui-pipe.py` (tool-calling / conversational / compaction) toggled by tool state.
- Ollama code paths no longer wanted.
- Two 1,700+ line files (`api/server.js`, `tools/executor.js`) mixing unrelated concerns.
- Basic auth used timing-unsafe compares, dashboard password was echoed at startup, no CSRF on destructive endpoints, `debug.log` grew unbounded regardless of `DEBUG_MODE`.
- Dead schema views, dead DB methods, dead tool helpers, dead npm scripts.

## Root Cause
Organic growth. The early architecture (two processes + many model roles + Ollama support) stayed in place while features accumulated around it.

## Old Behaviour
- `src/index.js` spawned two child processes. Each created its own `DatabaseManager` with independent sql.js state. Dashboard writes collided with API writes; `_mergeExternalSettings` papered over it by mtime-compare.
- Every dashboard endpoint called `db.reload()` to re-parse the SQLite file (~5 reads per page load).
- Chat endpoint had a ~130-line streaming path and a near-duplicate ~130-line non-streaming path. Each provider had `chatCompletion` + `chatCompletionStream` with duplicated tool loops.
- `owui-pipe.py` had three model valves (`TOOL_CALLING_LLM_*`, `CONVERSATIONAL_LLM_*`, `COMPACTION_LLM_*`) and swapped models based on whether any tool was enabled.
- `debug.log` truncated on startup and written unconditionally. Dashboard startup printed `admin:change_this_password` in plain text.
- Basic auth used `===`; file-recall admin auth used `!==`.
- `package.json` listed nine `test:*` scripts pointing at a `tests/` directory that did not exist, a `db:init` pointing at a missing `init.js`, and `dashboard`/`dashboard:dev` that launched the dashboard as a standalone process.

## New Behaviour
- Single Node process. `src/index.js` creates one `DatabaseManager`, imports `createApiApp(db)` and `createDashboardApp(db)` factories, and binds each to its own port so existing reverse-proxy and firewall topologies keep working. No cross-process merge, no `db.reload()` anywhere.
- `src/database/instance.js` exports the singleton DB.
- One `streamChat({ provider, ... })` entrypoint in `src/tools/providers/index.js` — `PROVIDERS = { openai, anthropic }`. Adding a provider is drop-a-file-and-register.
- Chat endpoint has one code path; stream/non-stream is a branch at the emit sites. Temp-proxy image cleanup runs in `finally` so errors outside the LLM call no longer leak files.
- `owui-pipe.py` exposes `LLM_PROVIDER` + `LLM_MODEL` for chat and `COMPACTION_PROVIDER` + `COMPACTION_MODEL` for summarisation. Ollama removed.
- `debug.log` is only written when `DEBUG_MODE=true`. Console messages always fire so supervisor log rotation still works.
- Dashboard basic-auth, API bearer-auth and file-recall admin-auth all use `crypto.timingSafeEqual`. Startup no longer prints credentials.
- Dashboard adds a `Sec-Fetch-Site` middleware: any value other than `same-origin` or `none` is rejected on `POST`/`PUT`/`DELETE`/`PATCH`. Same-origin AJAX works unchanged.
- Sandbox manager gains `cleanupOrphans()`; `src/index.js` calls it at boot so stale `sandbox-*` containers from a crashed run do not linger.
- `.env.example` grouped into clear sections with no default values in secret-shaped fields.
- `package.json` has only `start` and `dev` scripts — all dead test scripts removed, `ollama` keyword removed.

## Key Design Decisions

1. **Two ports, one process.** Merging the dashboard into a path prefix on port 3000 would have changed the firewall/proxy story for existing deployments. Binding two Express apps from one process in `index.js` gives the shared DB without that migration burden.

2. **Factory-returning modules over top-level side effects.** `createApiApp(db)` / `createDashboardApp(db)` make the shared DB explicit at the call site and keep `index.js` as the one place that owns lifecycle (boot, shutdown, sandbox orphan sweep).

3. **CSRF via `Sec-Fetch-Site` rather than tokens.** Modern browsers always set this header on fetch/XHR. Cross-site fetches set `cross-site`; direct navigation sets `none`; our dashboard JS sets `same-origin`. No client change, no token plumbing, one middleware covers the whole dashboard API.

4. **Provider abstraction is a map, not a class hierarchy.** `const PROVIDERS = { openai, anthropic }` keeps the seam small. Each provider exports `streamChat` and nothing else.

5. **Executor split by category.** Web / images / memory / sandbox / date-time / file-recall are sibling files sharing a `{ result, sources, error }` return shape via `src/utils/tool-result.js`. The dispatcher is a `HANDLERS` map, matching the provider pattern.

6. **Keep `compaction_count` column.** The schema still has it so existing DBs upgrade without a destructive `ALTER TABLE`. Writes simply stopped.

7. **Timing-safe compare everywhere credentials are checked.** API bearer, dashboard basic, and file-recall admin all converged on the same `Buffer.from(...) + length check + crypto.timingSafeEqual` pattern. Uniform style removes the risk of one path being silently weaker than the others.

8. **Integration test harness removed after use.** `scripts/test-pipe.js` was written to drive SSE end-to-end without touching Open WebUI during the refactor. Once the refactor was verified it was deleted rather than kept as a stale harness that would drift from the pipe's payload shape.
