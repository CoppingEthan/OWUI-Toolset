# Title: Cost calculator fix + first unit tests + OpenAI/Ollama residue cleanup

## Issue Summary
Three things in one pass:
1. **Bug fix**: `calculateCost()` in `src/api/server.js` had a stale `provider = 'openai'` fallback branch left over from before the local-first refactor. Any model name containing `'gpt'` that wasn't `gpt-oss` would be priced against a deleted provider.
2. **Test coverage**: The repo had no `tests/` directory despite `package.json` referencing six `test:*` scripts. The two most load-bearing bits of the local-first refactor — the OpenAI CC → Anthropic message-format conversion used on escalation, and `calculateCost` (including the `"gpt-oss-20b → claude-sonnet-4-6"` combo string) — were only manually verified.
3. **Residue cleanup**: The 3df3de5 refactor removed the OpenAI and Ollama providers but left stale references in schema defaults, a duplicated cost-calc in the dashboard server, dashboard UI copy, and two reference-doc directories.

## Root Cause (for bug fix)
When OpenAI support was ripped out of `src/tools/providers/`, the provider-detection fallback in `calculateCost` wasn't updated. It still set `provider = 'openai'` for any model name containing `'gpt'`, then looked up OpenAI-specific pricing/multipliers. Not a crash (the DB-cost lookup tolerates a missing key and falls back to a default `{ input: 1, output: 3 }`), but silently wrong pricing.

## Old Behaviour
- `calculateCost` lived inline in `src/api/server.js` at lines 1516-1631 and couldn't be imported without booting the full server (app.listen + DB init at module load).
- Unknown `gpt*` models were routed through a deleted OpenAI provider path and an OpenAI-specific `input_tokens - cache_read_tokens` adjustment that no longer matches any real usage format.
- `convertToAnthropicFormat` lived inside `src/tools/providers/llama-server.js`, which transitively imports `executor.js` → `llama-manager.js` → `ssh2`. Could not be imported from a test file on a workstation without the full prod dep chain.
- `package.json` referenced six `test:*` scripts pointing at a `tests/` folder that didn't exist.
- `src/dashboard/server.js` had its own `calcCost` function (~45 lines) duplicating the pricing logic in `server.js`, with the same OpenAI residue plus an extra gap: it didn't handle the `"gpt-oss-20b → claude-sonnet-4-6"` combo string, so "Recalculate All Historical Costs" would misprice escalated requests.
- `src/database/schema.sql` seeded OpenAI GPT-5/5.1/5.2 cost entries, Ollama cost entries, and OpenAI cache-multiplier entries on fresh DBs.
- `src/database/database.js:getCacheMultipliers()` returned both `anthropic` and `openai` blocks.
- Dashboard HTML had a `.model-badge.openai` CSS class, model-label map entries for `gpt-5`/`gpt-5.1`/`gpt-5.2`/`ollama`, a cost-pattern input placeholder referencing gpt-5, help text mentioning gpt-5/ollama, and a model-class fallback that returned `'openai'` for anything non-Claude.
- `docs/openai/` and `docs/ollama/` held old provider reference docs for providers that no longer exist.

## New Behaviour
- `calculateCost` is a pure function in `src/utils/cost-calculator.js`. Takes `dbCosts` and `cacheMultipliers` as injectable params. `server.js` keeps a thin wrapper so every existing call site is unchanged. Unknown non-Claude models default to llama-server (free). All OpenAI-specific branches (input-token subtraction, cache multiplier default, gpt-5 hardcoded pricing) are gone.
- `convertToAnthropicFormat` moved to `src/utils/anthropic-format.js` as a dep-free pure utility. `llama-server.js` imports it from there.
- `tests/` exists with `node:test`-based suites: **26 tests, 26 passing in ~90ms**, zero new dependencies.
  - `tests/test-cost-calculator.js` covers local-model short-circuit, the escalation combo string, unknown-model fallback, dbCosts exact/family/default matching, cache read/write multipliers, explicit-provider override, and hardcoded fallback pricing when dbCosts is null.
  - `tests/test-format-conversion.js` covers system extraction, multi-system merge, base64/http image blocks, tool_call↔tool_use with id preservation, JSON and object-form argument parsing, tool_result grouping, consecutive-same-role merging, and a full escalation round-trip.
- `npm test` uses `node --test` directly — no test-runner framework.
- Dashboard's `calcCost` is gone; the recalculate-costs endpoint imports the same `calculateCost` function as the API server. Escalated historical requests now recalculate correctly.
- Schema seeds only Anthropic model costs and Anthropic cache multipliers on fresh DBs.
- `getCacheMultipliers()` returns `anthropic` only.
- Dashboard UI: `.model-badge.llama-server` replaces `.model-badge.openai`; label map carries `gpt-oss-20b`; help text and placeholder reference current models; class fallback returns `'llama-server'`.
- `docs/openai/` and `docs/ollama/` directories removed.

## Key Design Decisions
- **Extracted to `src/utils/` rather than exporting from `server.js`**: `server.js` calls `app.listen()` and initialises the SQLite database at module-load time, so any test importing from it would bind a port and hit the real DB. A pure utility file avoids this entirely and keeps `cost-calculator.js`/`anthropic-format.js` as the testable seam. Same rationale — different file — for the format-conversion extraction.
- **Injected `dbCosts`/`cacheMultipliers` rather than importing the DB**: keeps the function pure, side-effect-free, and fast to test. The server-side cache helpers (`getModelCostsFromDB`, `getCacheMultipliersFromDB`) stay where they are, and both `server.js` and `dashboard/server.js` pass their already-fetched values in.
- **Folded the dashboard's `calcCost` into the shared calculator**: the two implementations had drifted (dashboard version missing escalation combo handling; api version had OpenAI input-token subtraction dashboard didn't). Deduping locks them to one source of truth and fixes the dashboard's escalation-pricing gap for free.
- **Removed `'openai'` from `getCacheMultipliers` return**: the dashboard's POST /api/settings/cache-multipliers endpoint still accepts arbitrary `provider` keys and writes them as settings, so an admin can technically persist OpenAI multipliers — but `calculateCost` never reads them. The GET endpoint now matches the calculator's view of the world.
- **`node:test` over Vitest/Jest**: built into Node ≥18, no new dependency, no framework learning curve, parallel-safe. The existing `package.json` engines field already requires Node ≥18, so no upgrade needed.
- **Kept `isMultimodalContent` in `src/transformers/`**: the 2026-03-12-15-30 scratchpad explicitly documented the decision to keep that one helper after purging the rest of the transformers. Didn't re-litigate.
- **Left `src/api/server.js:1197, 1378` "OpenAI cache tokens" branches alone**: they read `prompt_tokens_details.cached_tokens`, which llama-server's OpenAI-compatible endpoint may or may not surface. Defensive code that costs nothing — removing it could hide a real token source if llama-server's usage shape evolves.
